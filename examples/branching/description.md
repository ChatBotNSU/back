# branching

Главный граф со счётчиком и ветвлением через `condition` — без сабграфов.

**Что демонстрирует:**

- Узел типа `set_variable` с операцией `+=` для инкремента счётчика. Если переменной нет в текущем скоупе, операция `+=` падает — поэтому в начале графа есть `n_seed_counter`, который однократно создаёт `counter = 0`. Этот шаг проходится один раз: после первого `send_message` граф возвращается на `n_read`, минуя `n_seed_counter`.
- `condition` сравнивает две переменные. Литералы как операнды не поддерживаются — поэтому пороговое значение лежит во вспомогательной переменной `thr`, которая создаётся отдельным `set_variable`.
- Все ветви `condition.branches` проверяются по порядку. Если ни одна не совпала — переход в `default_next_node_id`.

**Ожидаемое поведение:**

```
user > 1
bot  < Сообщение #1.0: ещё привыкаем друг к другу.
user > 2
bot  < Сообщение #2.0: ещё привыкаем друг к другу.
user > 3
bot  < Сообщение #3.0: уже хорошо знаем друг друга.
user > 4
bot  < Сообщение #4.0: уже хорошо знаем друг друга.
```
